# 심야 병동 맵·멀티플레이 방 스펙

- 문서 버전: 1.0
- 기준일: 2026-07-16
- 상태: 서버 구현 기준
- 코드 기준점: `src/shared/map.ts`, `src/server/engine.ts`, `src/client/game/ThreeGameView.ts`

## 1. 맵 토폴로지

- 맵 전체 크기는 75×58 타일이다.
- 3타일 폭 세로 복도 5개와 가로 복도 4개가 교차해 큰 사각형 루프와 우회 경로를 만든다.
- 방은 복도 사각 셀 안에 3행×4열, 총 12개를 생성한다.
- 방의 문 방향은 상·하·좌·우 중 시드 기반으로 선택한다. 방 바닥과 복도 사이에는 문 한 칸만 통로로 남겨 벽 통과를 막는다.
- 플레이어·귀신 A* 경로는 같은 `walkable` 타일 집합을 사용한다. 모든 방의 침대에서 리스폰 구역까지 문을 포함한 경로가 존재해야 맵을 승인한다.

## 2. 개인 플레이 방

- 방 바닥은 최소 20칸, 최대 25칸이다.
- 4×5, 5×4, 5×5, 6×4와 모서리를 제거한 꺾인 형태를 섞는다.
- 방마다 침대 1개를 두고 나머지 19~24칸을 건설 가능 타일로 사용한다.
- 한 방에는 한 명만 머문다.

## 3. 멀티플레이 공유 방

- 모든 방은 정확히 25칸이다.
- 방마다 서로 떨어진 침대 2개를 두고 23칸을 건설 가능 타일로 사용한다.
- 한 방 최대 인원은 침대 수와 같은 2명이다. 세 번째 플레이어는 다음 빈 침대로 배정한다.
- 같은 방의 두 플레이어는 빈 타일에 각각 건설할 수 있고, 누가 설치했는지와 관계없이 포탑·문·발전기 등 설비를 강화할 수 있다.
- 각 플레이어는 자기 침대만 강화할 수 있다.
- 침대 골드는 해당 침대 점유자에게만 지급한다.
- 발전기 전력은 `BuildingState.ownerId`와 플레이어 ID가 같은 발전기에서만 지급한다.
- 다른 플레이어가 설비를 강화해도 설치 소유자는 바뀌지 않으므로 생산 귀속도 바뀌지 않는다.

## 4. 귀신 목표 선택

- 귀신은 살아 있는 플레이어가 있는 방만 목표 후보로 사용한다.
- 목표가 필요할 때 후보 중 하나를 균등 무작위로 선택하며 첫 방이나 최근 방을 고정 우선하지 않는다.
- 쌍둥이 귀신은 후보 방이 2개 이상이면 다른 쌍둥이가 선택한 방을 제외하고 고른다.
- 목표 방의 문이 살아 있으면 문을 공격하고, 문이 파괴되면 그 방의 살아 있는 플레이어 중 가까운 플레이어를 공격한다.

## 5. 렌더링·조작

- 복도 판정은 단일 가로 띠가 아니라 `corridorTiles` 전체를 사용한다.
- 복도망 전체에 샘플 조명을 분산한다.
- 벽 높이는 본체 0.68, 상단 캡 0.73 유닛으로 제한한다.
- 방 안의 모든 침대는 독립 모델로 표시하고 선택 ID는 `bed:{roomId}:{bedIndex}`를 사용한다.
- 카메라는 점유 전 플레이어 추적, 점유 후 드래그·회전·0.5×~2.0× 확대/축소 규칙을 유지한다.

## 6. 검증 조건

- 동일 시드는 동일 맵을 만든다.
- 방 12개, 개인 방 20~25칸, 멀티 방 25칸, 모드별 침대 수가 일치한다.
- 모든 이동 가능 타일이 하나의 연결 성분이다.
- 각 방의 침대→플레이어 스폰 경로는 해당 방 문을 반드시 지난다.
- 멀티 두 명이 같은 방의 다른 침대에 배정되고, 공동 강화와 개인 생산 귀속이 서버 테스트로 검증된다.
- 쌍둥이는 서로 다른 점유 방이 존재할 때 서로 다른 목표를 선택한다.

# 심야 병동 캐릭터·커스터마이징 스펙

- 문서 버전: 1.9
- 기준일: 2026-07-24
- 상태: 세트 스킨 모델 구현 기준
- 코드 기준점: `src/shared/customization.ts`, `src/server/auth.ts`, `src/client/main.ts`, `src/client/game/ThreeGameView.ts`

## 1. 현재 구현 모델

- 캐릭터 카탈로그에는 `paperdoll`의 기본 콘셉트 이미지 12개만 둔다.
- 외형은 모자·옷·신발·장신구를 따로 합성하지 않는 **완성 세트 스킨**이다. 각 스킨은 해당 캐릭터의 앞·옆·뒤·걷기·수면 프레임을 한 아틀라스로 보유해 홈, 상점, 커스텀, 인게임에서 같은 실루엣을 사용한다.
- 스킨은 기본 지급하지 않으며 해당 기본 캐릭터를 보유한 계정만 구매할 수 있다. `탐험가 모모`만 이벤트가 `100P`, 나머지 완성 스킨은 `2,500P`다. 현재 일반 스킨은 원본 캐릭터 특성의 1.5배를 사용하고, 필요하면 `traitOverride`로 독자 능력을 정의한다. 실제 결제 상품 전환 시에도 이 구매 권한 모델을 그대로 사용한다.
- `탐험가 모모`는 기본 토끼와 달리 침대 점유 전 이동속도 ×1.5라는 독자 능력을 사용한다.
- 상점과 커스텀은 `캐릭터 / 스킨` 탭을 사용한다. 포탑 스킨은 추후 별도 카탈로그가 준비될 때까지 표시하지 않는다.

## 2. 목표

1. 플레이어는 실사형 귀신과 대비되는 작고 귀여운 2D 탑다운 생존자 스프라이트로 보인다.
2. 다른 게임의 캐릭터를 복제하지 않고 토끼·고양이·강아지·곰·여우·햄스터를 모티브로 한 독자적인 투명 PNG 세트 스킨을 만든다.
3. 승리 → 커스텀 포인트 획득 → 아이템 구매 또는 등급 해금 → 착용 → 다음 게임에서 표현되는 성장 순환을 만든다.
4. 기본 토끼를 제외한 생존자 캐릭터는 한 가지 서버 판정 특성을 가진다. 스킨은 해당 캐릭터의 특성을 기본적으로 1.5배 적용하거나, `traitOverride`로 독자 능력을 사용한다.

## 3. 화면 흐름

```text
게임 홈
├─ 플레이 방식 모달 → 혼자하기/친구랑하기/랭크전 선택
├─ 상점 → 꾸미기 영구 구매·전술 보급 수량 구매
└─ 내 보관함 → 보유한 캐릭터/스킨 착용 또는 스킨 해제
```

- 게임 홈 하단은 `상점 / 스테이지 / 커스텀`의 세 메뉴로 고정한다. 스테이지 화면에는 플레이 방식 선택 상자와 주 `스테이지 시작` 버튼을 나란히 둔다.
- 플레이 방식 상자는 네이티브 셀렉트 대신 혼자하기·친구랑하기·랭크전 설명과 초대 코드 참여를 포함한 모달을 연다. 스테이지 메뉴는 현재 일반 모드에서 해금된 스테이지 선택 모달을 열고, 랭크전에서는 활성 계약 목록을 연다.
- 플레이 선택 화면은 거대한 포스터와 번호 장식을 제거하고 혼자하기, 친구랑하기, 초대 참가, 랭크전을 명확한 기능 영역으로 나눈다.
- 상점은 `꾸미기`와 `전술 보급`을 분리한다. 꾸미기는 미보유 외형까지 전부 보여 주고 포인트로 영구 구매하며, 구매 후 자동 착용하지 않고 커스텀 탭에서 선택한다. 전술 보급은 수량 단위로 구매하는 비영구 소모품이며, 상세 규칙과 카탈로그는 `docs/spec/shop-consumables.md`를 따른다.
- 내 보관함은 기본 지급, 포인트 구매 완료, 달성한 등급 보상을 포함해 현재 계정이 사용할 수 있는 보유품만 보여 준다. 구매 버튼은 표시하지 않고 착용만 처리한다.
- 두 화면은 같은 2D 프리뷰와 `앞/옆/뒤` 버튼을 사용한다. 진입 시 `앞`이 기본 선택이며, 선택한 캐릭터 또는 스킨의 프레임만 교체한다.
- 카드를 누르면 프리뷰만 임시 전환하고 구매·착용 API는 별도 버튼을 누를 때만 호출한다. 스킨을 해제하면 기본 캐릭터 프레임으로 즉시 돌아간다.
- 모바일 검증 해상도는 세로 390×844와 가로 844×390이다. 세로에서는 피팅룸과 스크롤 가능한 보유품·상점 목록을 상하로 배치한다.

## 4. 캐릭터와 세트 스킨

| 구분 | 기본 지급/구매 | 표현 방식 |
| --- | --- | --- |
| 캐릭터 | 토끼 모모만 기본 지급, 나머지는 포인트 구매 | `paperdoll` 기본 콘셉트와 기본 이동·수면 프레임 |
| 스킨 | 모두 별도 구매, 탐험가 모모 100P·나머지 2,500P | 캐릭터별 완성형 앞·옆·뒤·걷기·수면 프레임 |

개별 모자·옷·장신구·신발·포탑 외장 슬롯과 런타임 합성은 사용하지 않는다. 포탑 스킨은 향후 별도 카탈로그가 준비될 때만 다시 도입한다.

시즌 순위로 지급하는 한정 스킨도 같은 완성형 세트 구조를 사용한다. 한정 스킨의 고유 특성은 혼자하기·친구랑하기에서 적용하되 랭크전에서는 계약 표준 능력치로 보정하고 외형만 유지한다.

### 생존자 고유 특성

| 캐릭터 | 특성 |
| --- | --- |
| 달고양이 루루 | 모든 포탑 공격속도 +15% |
| 구름강아지 몽 | 침대 점유 중 초당 골드 +1 |
| 도토리곰 밤이 | 모든 포탑 피해 +10% |
| 별여우 초롱 | 한 판 램프 사용 횟수 +1 |
| 유령햄스터 콩 | 처음 건설하는 수호 포탑이 Lv.2로 시작 (스킨 Lv.3) |
| 늪악어 크로크 | 모든 포탑 피해 +35% |
| 달오리 꽥 | 침대 점유 중 초당 골드 +3 |
| 달호랑이 라온 | 모든 수호 포탑 사거리 +1칸 |
| 별공룡 라그 | 모든 포탑 공격속도 +40% |
| 달원숭이 몽키 | 한 판 램프 사용 횟수 +2 |
| 요새고릴라 콩 | 방을 점유하면 문이 Lv.2가 됨 (스킨 Lv.3) |

## 5. 해금 방식

- `starter`: 계정 생성 즉시 사용 가능하다.
- `points`: 표시된 커스텀 포인트를 지불해 계정에 영구 귀속한다.
- 완성 스킨은 모두 `points` 상품이다. 기본 캐릭터가 없으면 상점에서 `캐릭터 구매 필요`로 비활성화한다.
- 기존 카탈로그의 `rank` 조건은 시즌 랭크가 아니라 혼자하기·친구랑하기 일반 진행 등급을 뜻한다. 둘 중 높은 진행 등급이 요구 등급 이상이면 자동 해금한다.
- 시즌 순위 한정 스킨과 칭호는 랭크 시즌 보상 지급 기록으로 해금하며 포인트로 재구매할 수 없다.
- 포인트 구매 아이템과 등급 아이템을 동시에 요구하지 않는다.
- 이미 구매한 아이템 재구매와 보유하지 않은 포인트 아이템 착용은 서버가 거절한다.

## 6. 포인트 경제

승리한 판에만 아래 포인트를 지급한다.

```text
승리 포인트 = 80 + min(420, 스테이지 전역 인덱스 × 4)
패배 포인트 = 0
```

- 쉬움 1은 80포인트, 노말 5는 100포인트, 에픽 20은 320포인트, 레전더리 후반은 최대 500포인트다.
- 같은 `matchId + accountId` 결과는 한 번만 기록되므로 재연결이나 재전송으로 중복 지급되지 않는다.
- 기본 캐릭터 가격은 각 카탈로그의 포인트 가격을 사용한다. 세트 스킨은 `탐험가 모모`만 이벤트가 100P, 나머지는 2,500P다.
- 커스텀 포인트는 유료 재화가 아니며 현재 버전에서는 게임 클리어로만 획득한다.

## 7. D1 저장 구조

| 테이블                    | 주요 필드                                   | 설명                        |
| ------------------------- | ------------------------------------------- | --------------------------- |
| `account_customization`   | `account_id`, `custom_points`, `appearance` | 지갑과 현재 착용 JSON       |
| `account_cosmetics`       | `account_id`, `item_id`, `purchased_at`     | 포인트로 영구 구매한 아이템 |
| `account_turret_loadouts` | `account_id`, `skins`                       | 향후 포탑 스킨용 예약 데이터 |

- 기존 `accounts` 테이블은 변경하지 않는다.
- 두 테이블은 계정 삭제 시 함께 삭제되도록 외래키 `ON DELETE CASCADE`를 사용한다.
- 기본 아이템과 등급 해금 아이템은 구매 행을 만들지 않고 카탈로그 규칙으로 판정한다.
- 구매, 착용, 포인트 지급은 서버 권한이며 클라이언트가 잔액이나 보유 목록을 직접 결정하지 않는다.

## 8. 2D 캐릭터·스킨 원칙

- 기본 캐릭터와 스킨은 512px 정사각 투명 캔버스의 같은 중심 앵커를 공유한다. 앞·옆·뒤·걷기·수면 모든 프레임은 이 앵커를 보존해야 한다.
- 이동은 몸통 흔들기나 좌우 보간 없이 걷기 프레임만 느리게 교대한다. 홈 프리뷰는 정면 보행 프레임을 사용한다.
- 수면은 아래를 보는 서기 프레임이 아니라 옆으로 누운 별도 `sleep.png`를 사용한다.
- 프리뷰는 캐릭터의 실루엣이 잘리지 않도록 전신 중앙을 기준으로 배치하고, 꼬리가 있는 여우 등은 몸통 중심을 앵커로 삼는다.
- 스킨은 본체에 레이어를 덧씌우지 않는다. 각 완성 스킨 프레임을 통째로 표시해 홈·상점·커스텀·인게임에서 모자·옷·신발이 어긋나는 문제가 생기지 않게 한다.

## 9. 1차 완료 기준

- 신규·기존 계정이 커스텀 화면에 진입할 수 있다.
- 승리 시 포인트가 서버에 한 번만 지급된다.
- 캐릭터와 세트 스킨 구매·착용 상태가 D1에 저장된다.
- 멀티플레이 입장 시 서버가 검증한 착용 상태가 `PlayerState`에 포함된다.
- 로컬 플레이어·다른 플레이어·봇은 서버가 검증한 캐릭터 또는 세트 스킨 2D 프레임으로 표시된다.
- 미보유 캐릭터·스킨도 잔액 변화 없이 프리뷰할 수 있고, 앞·옆·뒤 버튼은 같은 앵커의 해당 방향 프레임을 표시한다.
- 모바일 390×844와 844×390에서 게임 홈, 플레이 방식 모달, 상점, 커스텀 화면이 겹치지 않는다.

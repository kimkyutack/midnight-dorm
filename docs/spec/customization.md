# 심야 병동 캐릭터·커스터마이징 스펙

- 문서 버전: 1.2
- 기준일: 2026-07-16
- 상태: 1차 수직 기능 구현 기준
- 코드 기준점: `src/shared/customization.ts`, `src/server/auth.ts`, `src/client/main.ts`, `src/client/game/ThreeGameView.ts`

## 1. 목표

1. 플레이어는 실사형 귀신과 대비되는 작고 귀여운 3D 생존자로 보인다.
2. 다른 게임의 캐릭터를 복제하지 않고 토끼·고양이·강아지·곰·여우·햄스터를 모티브로 한 독자적인 저폴리 캐릭터를 Three.js 지오메트리로 만든다.
3. 승리 → 커스텀 포인트 획득 → 아이템 구매 또는 등급 해금 → 착용 → 다음 게임에서 표현되는 성장 순환을 만든다.
4. 꾸미기 보상은 외형만 바꾸며 이동속도·공격력·자원 생산 같은 전투 능력에는 영향을 주지 않는다.

## 2. 화면 흐름

```text
게임 홈
├─ 플레이 → 개인/멀티 선택 → 게임
└─ 커스텀 → 캐릭터/모자/옷/장신구/신발/포탑 → 구매·착용 → 게임 홈
```

- 게임 홈은 `플레이`와 `커스텀`을 같은 하단 메뉴에 둔다.
- 플레이 선택 화면은 거대한 포스터와 번호 장식을 제거하고 개인전, 멀티전, 초대 참가를 세 개의 명확한 기능 영역으로 나눈다.
- 커스텀 화면 왼쪽에는 고정 배율의 실시간 Three.js 3D 피팅룸과 보유 포인트, 오른쪽에는 슬롯 탭과 아이템 목록을 둔다.
- 피팅룸 캐릭터는 자동으로 확대·축소하지 않는다. 한 손가락 또는 마우스 드래그와 `앞/옆/뒤` 버튼으로 360도 외형을 확인한다.
- 보유 여부와 포인트 잔액에 관계없이 캐릭터·모자·옷·장신구·신발 카드를 누르면 현재 캐릭터에 임시 적용한다. 구매·착용 API는 별도 버튼을 누를 때만 호출한다.
- 모바일 가로 최소 검증 해상도는 844×390이다. 목록은 화면 높이에 맞춘 작은 카드 그리드로 표시한다.

## 3. 슬롯과 기본 장착

| 슬롯   | 기본 아이템          | 역할                                             |
| ------ | -------------------- | ------------------------------------------------ |
| 캐릭터 | 밤토끼 모모          | 얼굴, 귀, 털 색과 실루엣                         |
| 모자   | 등급 대표 모자       | 최고 표시 등급에 따라 밀짚모자부터 광륜까지 변화 |
| 옷     | 별무늬 잠옷          | 몸통과 팔의 주 색상                              |
| 장신구 | 없음                 | 목도리, 가방, 브로치, 등불                       |
| 신발   | 병동 슬리퍼          | 발의 색상과 발광 여부                            |
| 포탑   | 종류별 기본 외장 4종 | 수호포·연사포·서리포·천둥포의 외형               |

카탈로그는 현재 생존자 외형 29종과 포탑 외형 12종, 총 41종을 제공한다. 포탑은 종류별로 3개 외장을 가지며 구매·장착한 외장은 다음에 설치하는 해당 종류 포탑부터 서버 상태의 `skinId`로 확정된다.

## 4. 해금 방식

- `starter`: 계정 생성 즉시 사용 가능하다.
- `points`: 표시된 커스텀 포인트를 지불해 계정에 영구 귀속한다.
- `rank`: 개인·멀티 중 높은 표시 등급이 요구 등급 이상이면 자동 해금한다.
- 포인트 구매 아이템과 등급 아이템을 동시에 요구하지 않는다.
- 이미 구매한 아이템 재구매와 보유하지 않은 포인트 아이템 착용은 서버가 거절한다.

## 5. 포인트 경제

승리한 판에만 아래 포인트를 지급한다.

```text
승리 포인트 = 80 + min(420, 스테이지 전역 인덱스 × 4)
패배 포인트 = 0
```

- 쉬움 1은 80포인트, 노말 5는 100포인트, 에픽 20은 320포인트, 레전더리 후반은 최대 500포인트다.
- 같은 `matchId + accountId` 결과는 한 번만 기록되므로 재연결이나 재전송으로 중복 지급되지 않는다.
- 초기 구매 가격은 신발 180, 장신구 200, 모자 250, 옷 350, 캐릭터 500포인트부터 시작한다.
- 커스텀 포인트는 유료 재화가 아니며 현재 버전에서는 게임 클리어로만 획득한다.

## 6. D1 저장 구조

| 테이블                    | 주요 필드                                   | 설명                        |
| ------------------------- | ------------------------------------------- | --------------------------- |
| `account_customization`   | `account_id`, `custom_points`, `appearance` | 지갑과 현재 착용 JSON       |
| `account_cosmetics`       | `account_id`, `item_id`, `purchased_at`     | 포인트로 영구 구매한 아이템 |
| `account_turret_loadouts` | `account_id`, `skins`                       | 포탑 종류별 현재 외장 JSON  |

- 기존 `accounts` 테이블은 변경하지 않는다.
- 두 테이블은 계정 삭제 시 함께 삭제되도록 외래키 `ON DELETE CASCADE`를 사용한다.
- 기본 아이템과 등급 해금 아이템은 구매 행을 만들지 않고 카탈로그 규칙으로 판정한다.
- 구매, 착용, 포인트 지급은 서버 권한이며 클라이언트가 잔액이나 보유 목록을 직접 결정하지 않는다.

## 7. 3D 캐릭터 원칙

- 머리 대 몸 비율은 약 1:1, 전체 키는 기존 인간 캐릭터의 약 72%로 줄인다.
- 큰 눈, 짧은 팔·다리, 둥근 볼과 동물별 귀·주둥이로 한눈에 캐릭터를 구분한다.
- 털은 약한 클리어코트, 눈은 강한 반사, 몸과 신발은 둥근 입체 지오메트리를 사용한다. 정면뿐 아니라 옆·뒤에서도 동물별 귀와 꼬리 실루엣이 구분되어야 한다.
- 모델 정면은 로컬 `-Z`로 통일하며 이동 회전은 이 축을 기준으로 계산해 뒤로 걷는 것처럼 보이지 않게 한다.
- 캐릭터가 침대를 점유하면 기존처럼 옆으로 눕되 작은 체형에 맞춰 이름표와 HP 바 높이도 낮춘다.
- 봇도 서로 다른 동물·옷·모자를 사용해 네 명이 같은 모델처럼 보이지 않게 한다.
- 모자·옷·장신구·신발은 각 슬롯이 독립적이며 전투 판정과 충돌 크기는 바꾸지 않는다.

## 8. 1차 완료 기준

- 신규·기존 계정이 커스텀 화면에 진입할 수 있다.
- 승리 시 포인트가 서버에 한 번만 지급된다.
- 포인트 아이템 구매, 등급 아이템 해금, 모든 슬롯 착용 상태가 D1에 저장된다.
- 멀티플레이 입장 시 서버가 검증한 착용 상태가 `PlayerState`에 포함된다.
- 포탑 외장은 D1에서 읽어 입장 헤더로 전달하고, 설치 시 `BuildingState.skinId`에 저장되어 모든 참가자에게 같은 모델로 보인다.
- 로컬 플레이어와 다른 플레이어, 봇의 귀여운 3D 캐릭터가 인게임에 표시된다.
- 미보유 포인트 아이템도 잔액 변화 없이 해당 아이템 클릭 시 3D 캐릭터에서 미리 볼 수 있다.
- 3D 피팅룸에서 앞·옆·뒤 버튼과 드래그 회전이 작동하며 캐릭터 배율은 고정된다.
- 모바일 844×390에서 게임 홈, 플레이 선택, 커스텀 화면이 겹치지 않는다.

# 심야 병동 비주얼·모바일 UX 스펙

- 문서 버전: 4.2
- 기준일: 2026-07-23
- 상태: 현재 클라이언트 구현 기준
- 코드 기준점: `src/client/styles.css`, `src/client/game/ThreeGameView.ts`, `src/shared/stageThemes.ts`, `public/assets/sprites`

이 문서는 게임 화면을 시네마틱 호러와 수직 탑다운 2D 조작 구조로 통일하기 위한 기준이다. 로비·티저는 프로젝트 전용 생성 이미지를 사용한다. 전장·설비·상호작용 효과는 Three.js 직교 카메라를 유지하고, 생존자와 본체 귀신은 같은 시점의 2.5D 투명 스프라이트로 단계적으로 교체한다.

## 1. 디자인 방향

1. 배경은 검정 한 색이 아니라 짙은 남색 레이어와 은은한 청록·보라 조명으로 깊이를 만든다.
2. 전장은 하늘에서 수직으로 내려다보는 직교 카메라를 사용해 방 안 타일이 중첩 없이 모두 보이게 한다.
3. 중요한 조작은 밝은 청록, 보조 조작은 라벤더, 비용·보상은 금색, 위험은 분홍빛 적색으로 일관되게 표시한다.
4. 정보량이 많은 HUD는 불투명 상자가 아니라 반투명 유리 패널로 구성해 전장을 가리지 않는다.
5. 장식보다 가독성을 우선한다. 색상만으로 상태를 구분하지 않고 아이콘, 게이지, 텍스트를 함께 사용한다.

## 3. 인터페이스 컴포넌트

- HUD와 모달은 16~26px 둥근 모서리, 얇은 밝은 테두리, 16~24px 배경 블러를 사용한다.
- 상단 자원 수치는 아이콘이 포함된 작은 캡슐 칩으로 표시한다.
- 우측 행동 버튼은 오래된 원형 게임패드 형태 대신 둥근 사각형 또는 캡슐 형태로 만든다.
- 주 행동 버튼은 금색, 가방은 반투명 남색, 현재 선택 상태는 청록 테두리로 구분한다. 선택 해제는 패널 바깥 터치로 처리해 별도 취소 버튼을 두지 않는다.
- 건설 패널은 설비 아이콘, 이름, 현재/다음 레벨, 효과, 골드·전력 비용을 한 카드 안에서 바로 읽을 수 있게 한다.
- 연결·재접속·게임 단계는 상단의 작은 상태 칩으로 표시하며 전투 영역을 덮는 큰 배너를 사용하지 않는다.
- 토스트는 하단 중앙에 표시하고, 정보·성공·위험 상태마다 테두리와 아이콘 색을 바꾼다.

## 4. 전장 오브젝트

- 방 바닥은 둥근 사각 타일과 낮은 대비의 내부 광택을 사용한다.
- 복도는 방보다 어둡게 하되 이동 가능 영역임을 알 수 있는 미세한 청색 패턴을 둔다.
- 벽은 개별 벽돌이 강하게 보이는 회색 블록 대신 하나의 매끄러운 남색 방벽처럼 이어 보이게 한다.
- 문은 벽 경계 타일 위에 낮게 누운 평면 패널로 표현해 방 안 타일을 가리지 않는다.
- 침대는 작은 매트리스 대신 청록 조명이 들어오는 수면 포드로 표현한다.
- 포탑과 발전기 등 설비는 원형 링과 중앙 코어 실루엣을 사용하고, 종류별 색·포신·방출 형태로 역할을 구분한다. 포탑은 Lv.1~3 기본형, Lv.4~6 장갑 고리, Lv.7~9 측면 축전기, Lv.10~12 다중 포신·방열판, Lv.13 이상 최종 코어·광륜의 다섯 외형 단계로 진화한다.
- 문은 10단계 재질을 사용한다: 나무 문, 녹슨 강철문, 빛바랜 강철문, 빨간 강철문, 단단한 철창, 빛나는 철창, 강철 티타늄, 은빛 티타늄, 금빛 티타늄, 다이아 티타늄. 이후 등급 확장 단계는 이 목록 뒤에 추가한다.
- 문 레벨이 올라가도 크기가 벽과 겹치지 않도록 시각 스케일 증가는 최대 32%로 제한한다.

## 5. 전투 피드백

- 포탑 포신은 현재 목표 방향을 바라보고 발사체 또는 레이저가 포탑에서 귀신까지 실제로 이동해야 한다.
- 피격 시 귀신의 색상 플래시, 체력 바 감소, 짧은 충격 파티클을 동시에 표시한다.
- 문에는 현재 HP와 최대 HP를 나타내는 게이지와 숫자를 항상 표시한다.
- 문의 긴 축은 방 경계 방향을 계산해 상하·좌우 출입구에 맞춘다.
- 비점유 방은 평면 문 패널을 옆으로 줄여 열린 상태로 보이고, 침대 점유 후 중앙으로 펼쳐 닫는다. 파괴된 패널은 다시 표시하지 않는다.
- 업그레이드 가능한 건물은 건물 중앙에 겹치는 작은 노란 원형 테두리와 굵은 위쪽 화살표를 표시한다. 재화가 부족하면 회색으로 바꾸되 형태는 유지한다.
- 귀신이 HP 20% 이하에서 후퇴하면 이동 방향 표시와 후퇴 상태 라벨을 함께 표시한다.
- 문 파괴, 플레이어 사망, 귀신 처치처럼 결과가 바뀌는 사건은 화면 흔들림과 색상 플래시를 짧게 사용하되 반복 공격에는 남용하지 않는다.
- 침대 골드가 정수 단위로 증가하면 침대 위에서 금색 `◆ +n` 피드백이 떠오른다. 팝업은 깊이 검사를 끄고 오브젝트보다 높은 렌더 순서를 사용해 벽이나 캐릭터에 눌리지 않는다.
- 발전기 또는 아이템 전력이 정수 단위로 증가하면 대표 발전기 위에서 청록색 `⚡ +n` 피드백이 떠오른다.
- 자원 팝업의 512×128 텍스처와 스프라이트는 항상 4:1 비율을 유지한다. 상승·페이드 애니메이션도 초기 `x/y` 비율을 보존해 텍스트가 좌우로 눌리지 않게 한다.

## 6. 카메라·터치·이동감

- 침대 점유 전에는 카메라가 플레이어를 따라가며 드래그 입력을 받지 않는다.
- 침대를 점유한 순간부터 카메라 추적을 해제하고 한 손가락 드래그로 자유 이동할 수 있게 한다.
- 귀신에게 잡혀 사망하면 캐릭터는 옆으로 쓰러지는 모션으로 전환하고, 침대 점유 여부와 관계없이 카메라 자동 추적을 해제해 관전 이동을 허용한다.
- 카메라는 북쪽이 화면 위로 오는 수직 직교 시점으로 고정하고 회전하지 않는다.
- 세로 기본 시야는 가로 약 8.4타일을 보여 방 전체와 주변 동선을 함께 확인하게 한다.
- 확대는 기본 대비 최대 1.5배, 축소는 기본 대비 최대 2배인 `0.5×~1.5×` 범위로 제한한다.
- 모바일은 두 손가락 핀치와 하단 `− / +` 버튼으로 확대·축소한다. 비틀기와 회전 버튼은 제공하지 않는다.
- PC는 마우스 휠로 확대·축소하고 좌·우 드래그로 카메라를 평행 이동한다.
- 조이스틱 지름은 약 120px, 주요 터치 목표는 최소 48px을 유지한다.
- 세로 화면에서는 조이스틱을 숨긴다. 침대를 점유하기 전 캐릭터 본체를 누르고 드래그하면 화면 기준 드래그 방향을 월드 이동 벡터로 변환해 이동한다. 손을 떼거나 취소되면 즉시 정지한다.
- 세로 화면 좌측 상단에는 현재 캐릭터 얼굴 버튼을 표시한다. 버튼을 누르면 침대 점유 후 자유 카메라 상태에서도 내 캐릭터 위치를 다시 카메라 중심으로 이동시킨다.
- 침대를 점유하기 전에는 고정 침대 버튼을 노출하지 않는다. 빈 침대의 상호작용 범위에 들어오면 침대 옆에 `잠자기` 버튼을 투영하고, 범위를 벗어나거나 점유하면 즉시 숨긴다.
- 플레이어와 귀신의 방향 회전은 `+180°/-180°` 경계에서도 최단 각도 차이로 보간한다. 조이스틱을 360° 돌릴 때 경계에서 반대 방향으로 되감기지 않아야 한다.
- 플레이어 기본 속도는 초당 4.8타일, 기본 귀신 속도는 초당 3.55타일로 설정해 이동 대기 시간을 줄인다.
- 빈 타일 터치는 전역 300ms, 같은 타일은 520ms 동안 다시 선택되지 않는다. 설치를 시작하면 해당 패널의 모든 설비 버튼을 잠그고, 클라이언트와 서버가 각각 350ms 설치 입력 제한을 적용해 다른 설비가 연속 설치되는 것을 막는다.
- 이동 중인 내 캐릭터는 로컬에서 즉시 예측하고 서버 위치 오차가 0.9타일을 넘을 때만 완만하게 보정한다. 1.5타일 이상의 큰 오차는 빠르게 보정하되 모든 보정 이동에도 벽 충돌을 적용한다.
- 서버는 초당 20회 이동을 계산하고 초당 15회 스냅샷을 일정한 누적 주기로 전송한다. 이동 입력 자체에는 즉시 전체 스냅샷을 재방송하지 않아 불필요한 렌더 갱신을 줄인다.
- 플레이어와 귀신의 한 프레임 이동량을 최대 0.12타일 단위로 나누고 서버·클라이언트가 같은 원형 충돌 반경을 사용한다. 지연이나 프레임 드롭이 있어도 벽 타일을 건너뛸 수 없다.

## 7. 진입 흐름과 게임 홈

1. 최초 실행 시 6.7초 길이의 복도 추격 티저를 표시한다. 사용자는 우측 상단 `건너뛰기`로 즉시 종료할 수 있다.
2. 티저는 브라우저 프로필의 `openingSeen`에 기록해 최초 한 번만 자동 재생한다. E2E 모드에서는 재생하지 않는다.
3. 로그인된 사용자는 병동 전경, 계정·등급·포인트, 왕관 모양 랭킹 진입점이 있는 게임 홈으로 이동한다. 홈의 작은 3D 캐릭터는 이동 방향인 왼쪽을 바라보고 양팔을 뒤로 뻗은 채 귀신을 피해 오른쪽에서 왼쪽으로 반복해서 달린다. 홈 캐릭터는 이전 프리뷰의 75% 크기이며, 추격 귀신은 최종 캐릭터 높이의 약 1.5배로 표현한다. 추격자는 별도 아이콘이 아니라 인게임 `wanderer` 귀신과 같은 Three.js 모델을 사용하며, 긴 검은 머리·창백한 얼굴·붉은 눈·낡은 회백색 치마를 가진 처녀귀신 실루엣이어야 한다. 귀신은 진행 방향으로 양손을 뻗고 어깨를 번갈아 흔들며 붙잡으려 달린다. 캐릭터와 귀신은 하나의 추격 그룹으로 움직이되 서로 겹쳐 보이지 않는 간격을 유지하고, 화면 비율이 달라도 귀와 발이 프리뷰 영역에서 잘리지 않아야 한다.
4. 홈 하단은 코드로 직접 그린 대형 SVG 아이콘 세 개로 구성한다. 화면에는 텍스트 라벨을 노출하지 않되 각 버튼의 접근성 이름은 `상점 / 스테이지 / 커스텀`으로 유지한다. 스테이지 영역의 좌측 플레이 방식 상자는 싱글·멀티 선택 모달을 열고, 우측 `스테이지 시작` 버튼은 현재 모드와 스테이지로 즉시 방을 만든다.
5. 로그인 화면도 같은 병동 배경을 사용해 인증 화면과 게임 홈이 하나의 세계처럼 이어지게 한다.
6. 대기실은 `새벽조 편성` 화면으로 구성하고, 룸 코드·참가자·스테이지 HP/공격 배율·맵 규칙을 한 화면에서 확인한다.
7. 결과 화면은 생존 시간, 귀신 레벨, 스테이지, 포인트 보상을 분리한 `NIGHT REPORT` 카드로 표시한다.
8. PWA 화면 방향은 `any`로 선언하고 세로 회전 차단 오버레이를 사용하지 않는다. 홈·로비·커스텀·상점·전장은 390×844와 844×390 모두에서 조작 가능해야 한다.
9. 홈에서는 `심야 병동` 제목과 별도 나가기 버튼을 노출하지 않는다. 포인트는 가로·세로 모두 계정 카드와 같은 상단 행에 두고, 로그아웃은 설정 모달 안에서만 제공한다. 현재 스테이지 카드는 버튼으로 동작해 홈을 벗어나지 않고 해금된 난이도를 선택할 수 있어야 한다.
10. 세로 인게임의 설정 버튼은 우측 상단, 연결 핑은 그 왼쪽에 둔다. 침대 점유 전에는 침대 아이콘만, 점유 후에는 그 자리에 가방 아이콘만 표시하며 텍스트 라벨은 접근성 이름으로만 유지한다.
11. 세로 홈의 스테이지 요약은 보조 설명을 두 줄까지 자연스럽게 줄바꿈하며 말줄임으로 자르지 않는다. 세로 대기실은 접속자 목록이 컨테이너 너비를 사용하고, `선택 스테이지` 카드의 단계명은 카드 중앙에 정렬한다. 생존자 카드는 60px 높이의 컴팩트 행으로 표시하며, 상태색 원 대신 해당 플레이어가 장착한 캐릭터 얼굴을 표시한다.
12. 세로 인게임의 다른 생존자 얼굴 행은 골드·전력 아래에서 화면 안전 여백 전체 폭을 사용한다. 준비 시간에는 화면 중앙보다 약간 위에 남은 초를 큰 적색 숫자로 표시한다.
13. Android 브라우저가 `데스크톱 사이트` 호환 모드로 980px 가상 viewport를 강제하더라도, 터치 가능한 세로 화면은 390px 논리 viewport로 정규화한다. 세로 레이아웃 분기는 폭 상한이 아니라 `orientation: portrait`를 기준으로 하며 일반 데스크톱 포인터에는 이 보정을 적용하지 않는다.

| 등급   | 배지        | 플레이어 모자                |

| 초고수 | 왕관 표식   | 은빛 왕관                    |
| 베테랑 | 지휘관 별   | 황금 지휘관 왕관             |

- 배지는 계정 홈, 모드 선택, 대기실 참가자 카드, 인게임 HUD에서 등급명 왼쪽에 항상 표시한다.

- 인게임 이름표에도 배지 기호, 등급명, 닉네임을 함께 표시해 멀리서도 등급을 구분한다.
- 배지와 모자는 외부 이미지가 아니라 CSS와 Three.js 지오메트리로 렌더링한다.

## 10. 귀신과 생성 이미지 자산

- `opening-chase.webp`: 플레이어가 세 귀신에게 쫓기는 16:9 시네마틱 티저 배경이다.
- `dorm-home.webp`: 로그인·게임 홈·모드 선택에 공통으로 사용하는 16:9 병동 배경이다.
- `ghost-roster.webp`: 기본형, 질주귀, 거구, 봉인술사, 쌍둥이 A/B를 담은 초기 6종 콘셉트 시트다.
- `public/assets/sprites`에는 생존자 12종과 본체 귀신 9종의 투명 PNG 콘셉트·이동·공격 프레임이 있다.
- 생존자는 정면·후면·우측 방향별 대기 1장과 걷기 3장을 사용한다. 귀신은 같은 이동 프레임에 방향별 준비·타격·회복 공격 프레임 3장을 추가한다. 왼쪽은 우측 프레임을 수평 반전한다.
- 문틈 도약귀는 청록 포털 고리, 언데드는 녹색 병원복과 독성 발광, 거대 귀신은 큰 검은 연기 체형과 청록 눈으로 구분한다. 언데드 미니미는 9종 본체 수에 포함하지 않는 0.42배 소환체다.
- 스프라이트 런타임 교체가 끝날 때까지 기존 Three.js 귀신 모델을 폴백으로 유지하며, 서버의 이동·충돌·공격 판정은 변경하지 않는다.
- 모든 생성 이미지는 이 프로젝트용 원본 프롬프트로 만든 독자 디자인이며 다른 게임의 캐릭터나 화면을 복제하지 않는다.

## 11. 구현 원칙

- 다른 게임의 이미지, 아이콘, 캐릭터 실루엣을 복제하지 않는다. 장르의 정보 구조와 밸런스 원칙만 참고한다.
- DOM UI는 `styles.css`의 공통 색상과 패널 규칙을 사용하고, Three.js 재질과 조명도 같은 색상 계열을 사용한다.
- 서버 이동·충돌·A\* 경로는 기존 직교 타일 좌표를 유지한다. 판정계를 교체하지 않고 Three.js 직교 렌더링으로 표시만 수직 2D로 바꿔 멀티플레이 동기화 안정성을 보존한다.
- 새로운 설비나 귀신 변종을 추가할 때는 흑백 실루엣만 보아도 기존 종류와 구별되어야 한다.
- 모바일 가로 화면 844×390을 최소 검증 해상도로 삼고, HUD와 행동 버튼이 플레이 공간을 과도하게 가리지 않는지 확인한다.
- 벽 본체와 상단 캡은 수직 시점에서 한 타일 안에 정확히 들어오게 해 방 안 타일 터치 영역을 침범하지 않는다.

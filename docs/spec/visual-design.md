# 심야 병동 비주얼·모바일 UX 스펙

- 문서 버전: 3.3
- 기준일: 2026-07-22
- 상태: 현재 클라이언트 구현 기준
- 코드 기준점: `src/client/styles.css`, `src/client/game/ThreeGameView.ts`, `src/shared/stageThemes.ts`

이 문서는 게임 화면을 레트로 벽돌풍에서 시네마틱 호러와 원근 탑다운 3D 스타일로 통일하기 위한 기준이다. 로비·티저·귀신은 프로젝트 전용 생성 이미지를 사용하고, 전장·설비·캐릭터·상호작용 효과는 Three.js로 렌더링한다.

## 1. 디자인 방향

1. 배경은 검정 한 색이 아니라 짙은 남색 레이어와 은은한 청록·보라 조명으로 깊이를 만든다.
2. 전장은 원근 카메라, 입체 바닥·벽·설비, 동적 조명, 화면 가장자리 비네트로 탑다운 3D 깊이를 만든다.
3. 중요한 조작은 밝은 청록, 보조 조작은 라벤더, 비용·보상은 금색, 위험은 분홍빛 적색으로 일관되게 표시한다.
4. 정보량이 많은 HUD는 불투명 상자가 아니라 반투명 유리 패널로 구성해 전장을 가리지 않는다.
5. 장식보다 가독성을 우선한다. 색상만으로 상태를 구분하지 않고 아이콘, 게이지, 텍스트를 함께 사용한다.

## 3. 인터페이스 컴포넌트

- HUD와 모달은 16~26px 둥근 모서리, 얇은 밝은 테두리, 16~24px 배경 블러를 사용한다.
- 상단 자원 수치는 아이콘이 포함된 작은 캡슐 칩으로 표시한다.
- 우측 행동 버튼은 오래된 원형 게임패드 형태 대신 둥근 사각형 또는 캡슐 형태로 만든다.
- 주 행동 버튼은 금색, 가방은 반투명 남색, 현재 선택 상태는 청록 테두리로 구분한다. 선택 해제는 패널 바깥 터치로 처리해 별도 취소 버튼을 두지 않는다.
- 건설 패널은 설비 아이콘, 이름, 현재/다음 레벨, 효과, 골드·전력 비용을 한 카드 안에서 바로 읽을 수 있게 한다.
- 연결·재접속·게임 단계는 상단의 작은 상태 칩으로 표시하며 전투 영역을 덮는 큰 배너를 사용하지 않는다.
- 토스트는 하단 중앙에 표시하고, 정보·성공·위험 상태마다 테두리와 아이콘 색을 바꾼다.

## 4. 전장 오브젝트

- 방 바닥은 둥근 사각 타일과 낮은 대비의 내부 광택을 사용한다.
- 복도는 방보다 어둡게 하되 이동 가능 영역임을 알 수 있는 미세한 청색 패턴을 둔다.
- 벽은 개별 벽돌이 강하게 보이는 회색 블록 대신 하나의 매끄러운 남색 방벽처럼 이어 보이게 한다.
- 문은 갈색 목재문 대신 청록색 코어가 들어간 세로형 에너지 캡슐로 표현한다.
- 침대는 작은 매트리스 대신 청록 조명이 들어오는 수면 포드로 표현한다.
- 포탑과 발전기 등 설비는 원형 링과 중앙 코어 실루엣을 사용하고, 종류별 색·포신·방출 형태로 역할을 구분한다.
- 문 레벨이 15까지 올라가도 크기가 벽과 겹치지 않도록 시각 스케일 증가는 최대 32%로 제한한다.

## 5. 전투 피드백

- 포탑 포신은 현재 목표 방향을 바라보고 발사체 또는 레이저가 포탑에서 귀신까지 실제로 이동해야 한다.
- 피격 시 귀신의 색상 플래시, 체력 바 감소, 짧은 충격 파티클을 동시에 표시한다.
- 문에는 현재 HP와 최대 HP를 나타내는 게이지와 숫자를 항상 표시한다.
- 문의 긴 축은 방 경계 방향을 계산해 상하 출입구와 좌우 출입구에 각각 맞춰 회전한다. 좌우 벽의 문을 상하 벽 방향으로 고정하지 않는다.
- 비점유 방은 문 패널을 90도로 열어 두고, 침대 점유 상태가 서버 스냅샷에 반영되면 패널을 약 0.4초 동안 닫힌 각도와 중앙 위치로 보간한다. 파괴된 패널은 다시 표시하지 않는다.
- 귀신이 HP 20% 이하에서 후퇴하면 이동 방향 표시와 후퇴 상태 라벨을 함께 표시한다.
- 문 파괴, 플레이어 사망, 귀신 처치처럼 결과가 바뀌는 사건은 화면 흔들림과 색상 플래시를 짧게 사용하되 반복 공격에는 남용하지 않는다.
- 침대 골드가 정수 단위로 증가하면 침대 위에서 금색 `◆ +n` 피드백이 떠오른다. 팝업은 깊이 검사를 끄고 오브젝트보다 높은 렌더 순서를 사용해 벽이나 캐릭터에 눌리지 않는다.
- 발전기 또는 아이템 전력이 정수 단위로 증가하면 대표 발전기 위에서 청록색 `⚡ +n` 피드백이 떠오른다.
- 자원 팝업의 512×128 텍스처와 스프라이트는 항상 4:1 비율을 유지한다. 상승·페이드 애니메이션도 초기 `x/y` 비율을 보존해 텍스트가 좌우로 눌리지 않게 한다.

## 6. 카메라·터치·이동감

- 침대 점유 전에는 카메라가 플레이어를 따라가며 드래그 입력을 받지 않는다.
- 침대를 점유한 순간부터 카메라 추적을 해제하고 한 손가락 드래그로 자유 이동할 수 있게 한다.
- 기본 시야는 방 하나와 이웃 방의 절반 정도가 보이는 배율을 유지한다.
- 확대는 기본 대비 최대 2배, 축소는 기본 대비 최대 2배인 `0.5×~2.0×` 범위로 제한한다.
- 모바일은 두 손가락 핀치로 확대·축소하고 두 손가락 비틀기로 시점을 회전한다. 화면 하단 버튼으로도 같은 조작을 제공한다.
- PC는 마우스 휠로 확대·축소하고 우클릭 드래그 또는 하단 회전 버튼으로 방향을 바꾼다.
- 조이스틱 지름은 약 120px, 주요 터치 목표는 최소 48px을 유지한다.
- 플레이어와 귀신의 방향 회전은 `+180°/-180°` 경계에서도 최단 각도 차이로 보간한다. 조이스틱을 360° 돌릴 때 경계에서 반대 방향으로 되감기지 않아야 한다.
- 플레이어 기본 속도는 초당 4.8타일, 기본 귀신 속도는 초당 3.55타일로 설정해 이동 대기 시간을 줄인다.
- 빈 타일 터치는 전역 300ms, 같은 타일은 520ms 동안 다시 선택되지 않는다. 설치를 시작하면 해당 패널의 모든 설비 버튼을 잠그고, 클라이언트와 서버가 각각 350ms 설치 입력 제한을 적용해 다른 설비가 연속 설치되는 것을 막는다.
- 이동 중인 내 캐릭터는 로컬에서 즉시 예측하고 서버 위치 오차가 0.9타일을 넘을 때만 완만하게 보정한다. 1.5타일 이상의 큰 오차는 빠르게 보정하되 모든 보정 이동에도 벽 충돌을 적용한다.
- 서버는 초당 20회 이동을 계산하고 초당 15회 스냅샷을 일정한 누적 주기로 전송한다. 이동 입력 자체에는 즉시 전체 스냅샷을 재방송하지 않아 불필요한 렌더 갱신을 줄인다.
- 플레이어와 귀신의 한 프레임 이동량을 최대 0.12타일 단위로 나누고 서버·클라이언트가 같은 원형 충돌 반경을 사용한다. 지연이나 프레임 드롭이 있어도 벽 타일을 건너뛸 수 없다.

## 7. 진입 흐름과 게임 홈

1. 최초 실행 시 6.7초 길이의 복도 추격 티저를 표시한다. 사용자는 우측 상단 `건너뛰기`로 즉시 종료할 수 있다.
2. 티저는 브라우저 프로필의 `openingSeen`에 기록해 최초 한 번만 자동 재생한다. E2E 모드에서는 재생하지 않는다.
3. 로그인된 사용자는 병동 전경, 게임 타이틀, 계정 닉네임·개인/멀티 등급·혜택, 하단 `게임 시작` 버튼이 있는 게임 홈으로 이동한다.
4. `게임 시작`을 누른 다음에 개인/멀티 플레이와 스테이지를 고른다. 모드 선택 화면은 게시판형 단일 패널이 아니라 각각 번호와 설명을 가진 작전 포스터로 표현한다.
5. 로그인 화면도 같은 병동 배경을 사용해 인증 화면과 게임 홈이 하나의 세계처럼 이어지게 한다.
6. 대기실은 `새벽조 편성` 화면으로 구성하고, 룸 코드·참가자·스테이지 HP/공격 배율·맵 규칙을 한 화면에서 확인한다.
7. 결과 화면은 생존 시간, 귀신 레벨, 스테이지, 포인트 보상을 분리한 `NIGHT REPORT` 카드로 표시한다.

## 8. 등급 배지와 3D 모자

| 등급 | 배지 | 플레이어 모자 |
| ---- | ---- | ------------- |
| 하수 | 빈 마름모 | 낡은 밀짚모자 |
| 중수 | 채운 마름모 | 생존자 캡모자 |
| 고수 | 별빛 표식 | 야간 스냅백 |
| 초고수 | 왕관 표식 | 은빛 왕관 |
| 베테랑 | 지휘관 별 | 황금 지휘관 왕관 |
| 레전드 | 광휘 문장 | 심연의 전설 왕관과 발광 후광 |

- 배지는 계정 홈, 모드 선택, 대기실 참가자 카드, 인게임 HUD에서 등급명 왼쪽에 항상 표시한다.
- 3D 플레이어 머리 위 모자는 `PlayerState.displayRank`를 기준으로 서버가 검증한 최고 표시 등급과 일치해야 한다.
- 인게임 이름표에도 배지 기호, 등급명, 닉네임을 함께 표시해 멀리서도 등급을 구분한다.
- 배지와 모자는 외부 이미지가 아니라 CSS와 Three.js 지오메트리로 렌더링한다.

## 9. 스테이지별 3D 맵 테마

| 스테이지 티어 | 맵 테마 | 주요 환경 표현 |
| ------------- | ------- | -------------- |
| 쉬움·노말 | 폐쇄 정신병동 | 차가운 병실 바닥, 의료 링과 청록 조명 |
| 악몽 | 안개 낀 산림 | 이끼색 바닥, 나무와 짙은 녹색 안개 |
| 지옥 | 빙결 격리동 | 얼음 바닥, 청백색 결정과 냉기 안개 |
| 불지옥 | 붉은 사막 유적 | 사암 벽, 붉은 안개와 유적 기둥 |
| 에픽 | 폐기물 처리장 | 녹슨 상자, 드럼통과 탁한 황록 조명 |
| 신화 | 금단 의식 연구소 | 보랏빛 룬, 오벨리스크와 의식 조명 |
| 레전더리 | 공허의 성채 | 검은 첨탑, 분홍·보라 발광 고리 |

- 테마는 `stageId`의 티어로 결정되며 바닥, 방, 벽, 벽 상단, 침대, 조명, 안개, 리스폰 구역, 환경 장식에 함께 적용한다.
- 렌더러 캔버스의 `data-theme`에 현재 테마 ID를 기록해 브라우저 자동 검증이 가능해야 한다.
- 테마 오브젝트는 서버의 이동 가능 타일과 충돌 판정을 바꾸지 않는 장식 레이어다.

## 10. 귀신과 생성 이미지 자산

- `opening-chase.webp`: 플레이어가 세 귀신에게 쫓기는 16:9 시네마틱 티저 배경이다.
- `dorm-home.webp`: 로그인·게임 홈·모드 선택에 공통으로 사용하는 16:9 병동 배경이다.
- `ghost-roster.webp`: 기본형, 질주귀, 거구, 봉인술사, 쌍둥이 A/B를 담은 초기 6종 콘셉트 시트다.
- 귀신 시트는 순수 검정 배경을 `SCREEN` 합성해 배경을 제거하고, 각 변종을 512×432 프레임으로 분리한다.
- 실제 전장은 9종 본체 귀신을 Three.js 입체 모델로 구분한다. 문틈 도약귀는 청록 포털 고리, 언데드는 녹색 뼈 장식, 거대 귀신은 1.6배 이상 큰 체형과 쇠사슬을 사용한다. 언데드 미니미는 별도 본체 종류가 아닌 0.42배 소환체다.
- 모든 생성 이미지는 이 프로젝트용 원본 프롬프트로 만든 독자 디자인이며 다른 게임의 캐릭터나 화면을 복제하지 않는다.

## 11. 구현 원칙

- 다른 게임의 이미지, 아이콘, 캐릭터 실루엣을 복제하지 않는다. 장르의 정보 구조와 밸런스 원칙만 참고한다.
- DOM UI는 `styles.css`의 공통 색상과 패널 규칙을 사용하고, Three.js 재질과 조명도 같은 색상 계열을 사용한다.
- 서버 이동·충돌·A\* 경로는 기존 직교 타일 좌표를 유지한다. 판정계를 교체하지 않고 Three.js 원근 3D 렌더링과 회전·확대 가능한 탑다운 카메라를 사용해 멀티플레이 동기화 안정성을 보존한다.
- 새로운 설비나 귀신 변종을 추가할 때는 흑백 실루엣만 보아도 기존 종류와 구별되어야 한다.
- 모바일 가로 화면 844×390을 최소 검증 해상도로 삼고, HUD와 행동 버튼이 플레이 공간을 과도하게 가리지 않는지 확인한다.
- 벽 본체 높이는 0.68 유닛, 상단 캡은 약 0.73 유닛에 두어 원근 카메라에서도 방 안 타일을 직접 누를 수 있게 한다.
